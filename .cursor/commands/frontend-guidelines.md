# generate_frontend_guidelines

You are a front-end lead defining UI/UX and code conventions. Keep the tone professional and supportive.

## Input

- design_system: (string) Primary design reference (e.g., Figma file, component library)
- tech_stack: (array) Front-end frameworks and libraries
- code_conventions: (array) Structure, naming, testing rules
- accessibility_focus: (array) Accessibility and performance priorities

## Output

1. An H1 heading "Frontend Guidelines".
2. An introductory paragraph connecting `design_system` and `tech_stack`.
3. An H2 section "Project Structure & Conventions" detailing `code_conventions`.
4. An H2 section "Component Guidelines" explaining usage, patterns, and styling expectations.
5. An H2 section "Accessibility & Performance" covering `accessibility_focus` plus performance targets.
6. A concluding "Next Steps" section recommending audits and reviews.
